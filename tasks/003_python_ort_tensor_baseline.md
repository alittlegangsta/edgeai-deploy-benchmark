# Task 003

## Title

Establish the Python environment and inspect ONNX Runtime raw tensors.

## Status

Planned

## Batch

Batch A

## Dependencies

Task 002 (`Completed`).

## Recommended Branch

`feature/pc-batch-a-python-baseline`

## Recommended Commit

`feat(python): add ORT tensor baseline`

## Goal

Load the frozen ONNX model with Python ONNX Runtime, construct the exact model
input tensor, print actual model I/O, run one real inference, and preserve raw
output tensor statistics without implementing full detection postprocessing.

## Why This Task Exists

It separates environment, provider, model-loading, input construction, and raw
runtime-output problems from the later decoding and NMS logic.

## Knowledge Covered

- Reproducible Python runtime version capture.
- ONNX Runtime sessions and execution providers.
- Image-to-tensor layout and dtype construction.
- Runtime I/O metadata and finite-value statistics.
- Machine-readable evidence without fabricated values.

## Scope

Create the smallest Python package/application needed to validate the frozen
model in CPU ONNX Runtime. Implement deterministic image loading and the Task 002
input contract. Print and save actual input/output metadata and raw output shape,
dtype, minimum, maximum, mean, standard deviation, and finite-value counts.

Do not decode boxes, apply confidence thresholds, perform NMS, map coordinates,
draw detections, or benchmark performance.

## Allowed Files

```text
TASKS.md
tasks/003_python_ort_tensor_baseline.md
requirements/pc-runtime.txt
python/edgeai_benchmark/__init__.py
python/edgeai_benchmark/model_contract.py

data/samples/images/pc_reference.jpg
results/evidence/003/model_io.json
results/evidence/003/raw_tensor_stats.json
results/logs/003_ort_tensor_baseline.log
```

The frozen Task 002 manifest and ONNX model are read-only inputs.

## Forbidden Files

- Task 001 or Task 002 evidence and frozen model-contract fields.
- Full detection decoding, NMS, drawing, or benchmark code.
- C++ source and CMake files.
- Model binaries or downloaded samples.
- Package lockfiles generated by installing dependencies.
- Any file outside Allowed Files.

## Inputs

- Completed Task 002 model manifest and ignored local ONNX file.
- A small, licensed, user-supplied `pc_reference.jpg` with recorded provenance.
- An existing Python environment containing compatible Python, NumPy, OpenCV,
  ONNX Runtime CPU, and test dependencies.

Missing packages or sample/model files require a stop; Codex must not install or
download them.

## Expected Outputs

- A pinned `requirements/pc-runtime.txt` reflecting the verified environment,
  not a command that mutates it.
- A CLI that validates the manifest SHA, creates one FP32 NCHW batch tensor, and
  runs the frozen model with the CPU execution provider.
- Real JSON for runtime model I/O and raw tensor statistics.
- A real command log with provider and dependency versions.

## Implementation Requirements

- Validate the ONNX SHA256 against Task 002 before session creation.
- Request and report `CPUExecutionProvider`; stop if it is unavailable or if an
  unexpected provider is used.
- Read input/output names, shapes, and dtypes from the session and compare them
  to the frozen contract without adapting silently.
- Construct input using the frozen 640 by 640, batch-1 FP32 NCHW contract.
- Use deterministic letterbox infrastructure, BGR-to-RGB, HWC-to-CHW,
  `uint8`-to-`float32`, `[0,1]` normalization, contiguous storage, and a batch
  dimension. Full geometric validation belongs to Task 005.
- Save tensor metadata and aggregate numeric statistics only; do not store a full
  tensor dump or label any raw candidate as a detection.
- Reject non-finite input or output values and unknown output count/shape.
- Make output JSON deterministic except for explicitly labeled environment/time
  metadata.

## Build Commands

```bash

python3 -m py_compile \
  python/edgeai_benchmark/model_contract.py \
  python/edgeai_benchmark/preprocess.py \
  python/apps/ort_tensor_baseline.py
```

Version checks for `numpy`, `cv2`, and `onnxruntime` must be run read-only and
recorded before build acceptance.

## Run Commands

```bash
mkdir -p results/evidence/003 results/logs
PYTHONPATH=python python3 python/apps/ort_tensor_baseline.py \
  --model models/yolov5n-v7.0/yolov5n.onnx \
  --manifest models/yolov5n-v7.0/manifest.json \
  --image data/samples/images/pc_reference.jpg \
  --io-output results/evidence/003/model_io.json \
  --stats-output results/evidence/003/raw_tensor_stats.json \
  2>&1 | tee results/logs/003_ort_tensor_baseline.log
```

## Acceptance Criteria

- The existing environment and all relevant package versions are recorded.
- The model SHA matches Task 002 before loading.
- ONNX Runtime uses CPUExecutionProvider and loads the model successfully.
- Actual runtime I/O exactly matches the frozen Task 002 contract.
- The input is contiguous FP32 NCHW with batch 1 and spatial size 640 by 640.
- A real inference succeeds and every output value is finite.
- Actual raw output shapes, dtypes, and aggregate statistics are preserved.
- No NMS, full decoder, detections, drawing, or benchmark claim is introduced.
- Tests, JSON validation, and `git diff --check` pass.
- Only Allowed Files changed and no required command was skipped.

## Evidence to Preserve

Python/platform/package versions, provider list and selected provider, model hash,
input metadata, runtime I/O, output statistics, all command exit codes, tests,
attempt count, and final Git status.

## Automatic Retry Rules

Use at most three full repair loops for defects in Allowed Files. Stop immediately
for a missing/incompatible package, provider, model, sample, SHA mismatch, or
contract mismatch; do not modify the environment or frozen contract.

## Human Stop Conditions

All protocol environment/model/correctness/file-safety stops apply. Also stop if
the reference image provenance is unclear, output values are non-finite, or raw
output interpretation would require guessing.

## Codex Responsibilities

Validate every external input; keep raw tensor statistics distinct from decoded
detections; preserve true output; modify only Allowed Files; and update durable
state and blocking details.

## User Responsibilities

Provide the local environment and licensed reference image, resolve missing
dependencies, review runtime I/O/statistics, and decide whether anomalies are
acceptable.

## Known Risks

- Installed ONNX Runtime may use a different provider or version.
- OpenCV interpolation may later differ from C++ if parameters are not explicit.
- Raw statistics can look plausible even when preprocessing is wrong.
- A reference image may have EXIF orientation or color-profile surprises.

Report files, environment versions, provider, model-hash validation, input
metadata, actual model I/O, raw statistics, build/run/test outcomes, attempts,
skips, risks, and final Git status with real-versus-static labels.
