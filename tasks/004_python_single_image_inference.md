# Task 004

## Title

Implement Python ONNX Runtime single-image detection.

## Status

Planned

## Batch

Batch A

## Dependencies

Task 003 (`Completed`).

## Recommended Branch

`feature/pc-batch-a-python-baseline`

## Recommended Commit

`feat(python): add single-image ORT detection`

## Goal

Turn the verified raw ONNX output into a complete, transparent YOLOv5 single-
image detection pipeline with structured results, an annotated image, and
separate stage timings.

## Why This Task Exists

A readable Python reference defines preprocessing and postprocessing behavior
against which both native backends can later be compared.

## Knowledge Covered

- Letterbox geometry and color/layout conversion.
- YOLOv5 objectness and class-score decoding.
- Coordinate conversion, thresholding, NMS, and inverse mapping.
- Deterministic visualization and structured output.
- Stage-specific measurement boundaries.

## Scope

Implement one CPU ONNX Runtime CLI for one local image and the frozen Task 002
model. Complete preprocessing, inference, postprocessing, coordinate restoration,
clipping, drawing, JSON/text results, and separately labeled preprocess,
inference, postprocess, and visualization durations.

Do not add video support, C++ code, ncnn, formal benchmark aggregation, model
conversion, or a new export strategy.

- Any file outside Allowed Files.

## Inputs

- Completed Task 002 frozen model and manifest.
- Completed Task 003 Python package and verified environment.
- The same user-supplied reference image and its provenance.
- Explicit configuration for confidence threshold, IoU threshold, class-aware
  NMS, maximum detections, and 640 by 640 input.

## Expected Outputs

- A reusable, backend-independent Python postprocessor and visualizer.
- A CLI-produced annotated PNG.
- Structured JSON containing source/model hashes, thresholds, original and input
  geometry, ordered detections, and stage timings.
- A human-readable log generated by the same invocation.

## Implementation Requirements

- Letterbox with explicit scale, integer padding, interpolation, and padding
  color; preserve enough metadata for exact inverse mapping.
- Convert BGR to RGB, HWC to CHW, `uint8` to `float32`, normalize to `[0,1]`,
  make contiguous, and add batch dimension.
- Execute ORT only after model SHA and runtime contract validation.
- Parse the frozen raw output without assuming unverified names or dimensions.
- Compute class confidence as `objectness * class_score`; preserve objectness and
  selected class score separately in structured output.
- Convert `xywh` to `xyxy`, filter using the frozen confidence threshold, apply
  deterministic class-aware NMS with the frozen IoU threshold, and enforce the
  maximum-detection limit.
- Restore coordinates through the recorded letterbox scale/padding, clip to the
  original image bounds, and reject invalid boxes.
- Draw class, confidence, and rectangle without mutating the original used for
  tensor construction.
- Sort structured detections deterministically and use stable numeric formatting.
- Measure preprocess, inference, postprocess, and visualization separately with
  a monotonic clock. Treat these single-run timings as diagnostic, not formal
  benchmark evidence.
- Return nonzero for invalid arguments, missing files, SHA/contract mismatch,
  image decode failure, non-finite output, or output-write failure.

## Run Commands

```bash
mkdir -p results/images results/evidence/004 results/logs
PYTHONPATH=python python3 python/apps/ort_image.py \
  --model models/yolov5n-v7.0/yolov5n.onnx \
  --manifest models/yolov5n-v7.0/manifest.json \
  --config configs/yolov5n_v7_inference.json \
  --image data/samples/images/pc_reference.jpg \
  --output-image results/images/python_ort_reference.png \
  --output-json results/evidence/004/python_ort_detections.json \
  2>&1 | tee results/logs/004_python_ort_image.log

file results/images/python_ort_reference.png
```

## Acceptance Criteria

- Every required preprocessing transformation is implemented explicitly.
- Real ORT inference succeeds against the frozen model and contract.
- Objectness, class score, combined confidence, `xywh` conversion, thresholding,
  deterministic NMS, inverse mapping, and clipping are implemented and exercised.
- Every emitted box is finite, ordered, positive-area, and inside source bounds.
- Structured output contains real detections or explicitly records a real empty
  result without pretending that emptiness is correct.
- The annotated image is nonempty and decodable.
- Preprocess, inference, postprocess, and visualization timings are distinct and
  labeled diagnostic measurements.
- Invalid CLI/input cases return nonzero.
- Tests and `git diff --check` pass; only Allowed Files changed.
- A human reviews the image and detection plausibility before Task 005 proceeds.

## Evidence to Preserve

Model/image hashes, thresholds, original/letterbox geometry, real detections,
diagnostic stage timings, annotated image, command logs/exit codes, test output,
attempts, image-review decision, and final Git status.

## Automatic Retry Rules

Allow three complete repair loops for implementation defects. Stop immediately
for contract/SHA/environment mismatch or when detections are empty, malformed,
implausible, or require visual interpretation. Do not tune thresholds merely to
manufacture detections.

## Human Stop Conditions

Apply every protocol condition. Human review is mandatory for the annotated
image, class plausibility, clearly empty/abnormal results, and any proposed change
to thresholds, preprocessing, model contract, or output interpretation.

## Codex Responsibilities

Keep stages explicit, preserve real structured results, label timings honestly,
avoid benchmark claims, enforce error exits, record attempts, and stop for visual
approval.

## User Responsibilities

Confirm reference-image rights, visually inspect the annotated image, assess
detections and classes, and authorize any change to frozen thresholds or contract.

## Known Risks

- Letterbox rounding can shift restored boxes.
- Incorrect raw-output interpretation may yield plausible-looking detections.
- Threshold-edge boxes can make results sensitive to numeric variation.
- A single diagnostic duration is not representative performance.

## Completion Report Format

Report changed files, hashes/configuration, preprocessing geometry, real detection
summary, image path and visual-review state, stage timings labeled diagnostic,
build/run/tests, invalid cases, attempts, risks, and Git status.
